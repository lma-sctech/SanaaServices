# Checklist responsive

## Viewports

- [ ] 360 x 800.
- [ ] 390 x 844.
- [ ] 430 x 932.
- [ ] 768 x 1024.
- [ ] 1024 x 768.
- [ ] 1280 x 800.
- [ ] 1440 x 900.
- [ ] 1920 x 1080.
- [ ] Mobile paysage.
- [ ] Largeur intermediaire non couverte par un breakpoint exact.

## Integrite

- [ ] Aucun scroll horizontal involontaire.
- [ ] Aucun texte, bouton, badge ou image coupe.
- [ ] Les titres longs reviennent naturellement a la ligne.
- [ ] Le contenu reste lisible a 200 % de zoom.
- [ ] Les zones tactiles font au moins 44 x 44 px.
- [ ] Les actions ne collent pas au bord de l'ecran.
- [ ] Les contenus importants ne dependent pas du hover.

## Navigation

- [ ] Header desktop masque au bon moment.
- [ ] Menu mobile ouvrable et fermable au clavier et au toucher.
- [ ] Focus piege et restaure si le menu est modal.
- [ ] Ancres non masquees par le header sticky.
- [ ] Aucun changement de source entre menu desktop et mobile.

## Sections

- [ ] Hero lisible avec chaque image et chaque langue cible.
- [ ] Grilles passent de 1 a N colonnes sans largeurs fixes.
- [ ] Cartes de hauteurs differentes restent intentionnelles.
- [ ] Modales tiennent dans le viewport et scrollent correctement.
- [ ] Carrousels fonctionnent au doigt et au clavier.
- [ ] Footer reste comprehensible sur petit mobile.

## Formulaires

- [ ] Labels et erreurs visibles sans zoom.
- [ ] Clavier virtuel adapte au type de champ.
- [ ] Autofill n'endommage pas le style.
- [ ] Bouton d'envoi accessible lorsque le clavier est ouvert.
- [ ] Messages de succes et d'erreur ne debordent pas.

## Tres grands ecrans

- [ ] Largeur de lecture limitee.
- [ ] Espacement augmente sans diluer la hierarchie.
- [ ] Images conservent un point focal pertinent.
- [ ] Aucun bloc etire artificiellement sur toute la largeur.
