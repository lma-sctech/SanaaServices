# Checklist SEO

## Par route

- [ ] Title unique, descriptif et coherent avec l'intention.
- [ ] Meta description unique et factuelle.
- [ ] Un H1 principal.
- [ ] Hierarchie H2/H3 logique.
- [ ] URL stable et lisible.
- [ ] Canonical correct pour l'environnement de production.
- [ ] Open Graph et Twitter metadata.
- [ ] Image sociale disponible et dimensionnee.
- [ ] Contenu indexable rendu dans le HTML initial.

## Site

- [ ] `sitemap.ts` contient uniquement les routes publiques canoniques.
- [ ] `robots.ts` distingue production et previews.
- [ ] Pages privees, previews et espaces clients non indexables.
- [ ] Page 404 utile et statut correct.
- [ ] Aucun lien interne casse.
- [ ] Navigation et footer couvrent les pages importantes.

## Local SEO

- [ ] NAP identique sur toutes les pages et apps.
- [ ] Adresse, telephone et horaires verifies.
- [ ] Zone de service formulee sans keyword stuffing.
- [ ] `Organization` et type local adapte en JSON-LD.
- [ ] `sameAs` contient uniquement des profils officiels.
- [ ] Donnees structurees valides et conformes au contenu visible.
- [ ] Aucun faux avis ou aggregate rating.

## Internationalisation

- [ ] Une locale publiee est complete.
- [ ] `lang` correct sur le document.
- [ ] `dir="rtl"` pour l'arabe et interface testee en RTL.
- [ ] Hreflang reciproque entre versions equivalentes.
- [ ] Canonical reste dans la bonne locale.
- [ ] Slugs et metadata relus humainement.

## Contenu

- [ ] Aucune information TBD publiee.
- [ ] Dates et offres temporelles comportent une date de mise a jour.
- [ ] Pages reglementees validees par une personne qualifiee.
- [ ] Texte alternatif decrit la fonction de l'image, sans bourrage de mots-cles.
- [ ] Les CTA ont des libelles descriptifs.
